## Usage

```tsx showLineNumbers
import VideoRoot, {
  VideoViewport,
  VideoControls,
  VideoPlayTrigger,
  VideoSoundControl,
  VideoProgressBar,
  VideoPipTrigger,
  VideoFullscreenTrigger
} from "@/components/8starlabs-ui/video-player";

export function MyVideoPlayer() {
  return (
    <VideoRoot className="overflow-hidden rounded-xl">
      <VideoViewport src="https://vjs.zencdn.net/v/oceans.mp4" />

      <VideoControls className="flex items-center justify-between gap-4">
        <VideoPlayTrigger />
        <VideoSoundControl />
        <VideoProgressBar />
        <VideoPipTrigger />
        <VideoFullscreenTrigger />
      </VideoControls>
    </VideoRoot>
  );
}
```

## API Reference

### VideoRoot

Wraps the player in context and handles the control overlay visibility logic.

| Prop        | Type        | Default  | Description                                                    |
| ----------- | ----------- | -------- | -------------------------------------------------------------- |
| `className` | `string`    | -        | Additional classes for the outer player container.             |
| `children`  | `ReactNode` | Required | Player content, typically `VideoViewport` and `VideoControls`. |

Also accepts standard `div` props.

### VideoViewport

Renders the underlying HTML5 `<video>` element and syncs playback state into the player context.

| Prop        | Type     | Default  | Description                               |
| ----------- | -------- | -------- | ----------------------------------------- |
| `src`       | `string` | Required | Video source URL.                         |
| `className` | `string` | -        | Additional classes for the video element. |

Also accepts standard `video` props such as `poster`, `loop`, `muted`, `autoPlay`, and `controlsList`.

### VideoControls

Positions the custom controls in a bottom overlay that fades in on pointer movement.

| Prop        | Type        | Default  | Description                                    |
| ----------- | ----------- | -------- | ---------------------------------------------- |
| `className` | `string`    | -        | Additional classes for the controls wrapper.   |
| `children`  | `ReactNode` | Required | Control elements to render inside the overlay. |

Also accepts standard `div` props.

### VideoPlayTrigger

Play/pause button wired to the shared player state.

Accepts standard `button` props.

### VideoSoundControl

Mute toggle plus expandable vertical volume slider.

Accepts standard `button` props on the mute trigger.

### VideoProgressBar

Displays the current time, seek bar, and total duration. The seek bar supports hover preview and drag-to-seek interactions.

Accepts standard `div` props.

### VideoPipTrigger

Toggles Picture-in-Picture mode when supported by the browser.

Accepts standard `button` props.

### VideoFullscreenTrigger

Toggles fullscreen mode for the player container.

Accepts standard `button` props.

## Notes

- `VideoRoot` uses context internally, so all subcomponents must be rendered inside it.
- Controls appear on mouse movement and hide automatically after a short delay when the pointer is not over the controls.
- Clicking directly on the video toggles play and pause.
- The progress bar pauses playback while dragging and resumes playback afterward if the video was already playing.
- Picture-in-Picture support depends on the browser and platform.
- All components support standard HTML attributes and `className` overrides.
